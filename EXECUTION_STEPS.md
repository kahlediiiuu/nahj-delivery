# دليل التنفيذ الكامل - خطوة بخطوة بالتفصيل

كل خطوة هنا مكتوبة بحيث تعرف بالضبط: أي موقع تفتح، أي زر تضغط (بالعربي والإنجليزي كما سيظهر أمامك)، وماذا يظهر لك بعده.
نفّذها بالترتيب، ولا تقفز خطوة. إن ظهر لك شيء مختلف عمّا وصفته، توقف وأخبرني بالضبط ماذا ترى.

---

# المرحلة 1: إنشاء حساب Firebase (قاعدة البيانات)

1. افتح المتصفح (Chrome يُفضّل) واذهب إلى: `https://console.firebase.google.com`
2. سجّل الدخول بحساب Gmail الخاص بك (أنشئ واحداً إن لم يكن لديك عبر البحث في جوجل عن: **إنشاء حساب جيميل / create gmail account**)
3. ستظهر لك الصفحة الرئيسية، اضغط الزر الذي يظهر مكتوباً عليه **"Create a project"** (إنشاء مشروع)
4. اكتب اسم المشروع: `nahj-delivery` ثم اضغط **Continue** (متابعة)
5. سيسألك عن Google Analytics — اضغط **زر الإيقاف/التعطيل** (يمكنك تركه معطلاً، لسنا بحاجته) ثم **Create project**
6. انتظر حتى تظهر رسالة **"Your new project is ready"** ثم اضغط **Continue**

### الآن داخل المشروع:

7. من القائمة الجانبية اليسرى (☰)، ابحث عن قسم **Build**، اضغط عليه لتتوسع القائمة
8. اضغط **Realtime Database** → اضغط زر **Create Database**
9. اختر أقرب موقع خادم لك (مثال: لو أنت بالسعودية اختر أي خيار قريب مثل Europe أو Singapore) → **Next**
10. اختر **Start in locked mode** (وضع مغلق) → **Enable**
11. ستفتح صفحة قاعدة البيانات فارغة — هذا طبيعي، لاحظ الرابط بالأعلى، سيكون شبيهاً بـ:
    `https://nahj-delivery-default-rtdb.firebasedatabase.app` — احفظ هذا الرابط، ستحتاجه لاحقاً (هذا هو `FIREBASE_DATABASE_URL`)

12. ارجع للقائمة الجانبية → **Build** → **Firestore Database** → **Create database**
13. اختر **Start in production mode** → **Next** → اختر نفس الموقع السابق → **Enable**

### الحصول على مفتاح الاتصال (الأهم):

14. اضغط على ⚙️ (أيقونة الترس) بجانب "Project Overview" أعلى القائمة الجانبية → اختر **Project settings**
15. من الأعلى، اضغط تبويب **Service accounts**
16. اضغط الزر الأزرق **Generate new private key** → ثم تأكيد **Generate key**
17. سيُنزَّل ملف باسم شبيه بـ `nahj-delivery-firebase-adminsdk-xxxxx.json` إلى مجلد التنزيلات (Downloads) في جهازك
18. **احتفظ بهذا الملف جانباً، لا تشاركه مع أحد أبداً** — سنحتاجه في المرحلة القادمة

✅ **انتهت المرحلة 1.** أخبرني "انتهيت من المرحلة 1" لننتقل للتالية.

# المرحلة 2: رفع الكود إلى GitHub (لتشغيل الخادم لاحقاً)

GitHub هو موقع مجاني لتخزين الأكواد البرمجية، ونحتاجه لأن موقع تشغيل الخادم (Render) يطلب الكود من هناك.

1. اذهب إلى: `https://github.com` → اضغط **Sign up** (إنشاء حساب) وأكمل التسجيل ببريدك
2. بعد الدخول، اضغط الزر الأخضر **+** بالأعلى → اختر **New repository**
3. في خانة **Repository name** اكتب: `nahj-delivery`
4. اترك باقي الخيارات كما هي، اضغط **Create repository**
5. ستظهر صفحة فارغة فيها رابط الرفع، اضغط الرابط المكتوب **"uploading an existing file"**
6. الآن **افتح مجلد المشروع الذي أرسلته لك** على جهازك (بعد فك ضغط الملف nahj-delivery-system.zip)
7. اسحب مجلد `backend` بالكامل وأفلته داخل صفحة GitHub (Drag and drop)
8. انتظر حتى ينتهي الرفع (سترى شريط تقدم لكل ملف)، ثم اضغط الزر الأخضر بالأسفل **Commit changes**

✅ **انتهت المرحلة 2.** أخبرني "انتهيت من المرحلة 2".

# المرحلة 3: تشغيل الخادم (Backend) مجاناً على Render

1. اذهب إلى: `https://render.com` → اضغط **Get Started** وسجّل بحساب GitHub نفسه (زر **Sign up with GitHub**) — هذا يسهّل الربط تلقائياً
2. من لوحة Render، اضغط **New +** أعلى الصفحة → اختر **Web Service**
3. اختر مستودع (Repository) الذي رفعته: `nahj-delivery` → اضغط **Connect**
4. في الإعدادات التي تظهر:
   - **Name**: اكتب `nahj-backend`
   - **Root Directory**: اكتب `backend` (مهم جداً — لأن الكود داخل مجلد فرعي)
   - **Runtime**: تأكد أنه `Node`
   - **Build Command**: اكتب `npm install`
   - **Start Command**: اكتب `npm start`
   - **Instance Type**: اختر **Free**
5. انزل للأسفل إلى قسم **Environment Variables** (متغيرات البيئة) → اضغط **Add Environment Variable** وأضف كل سطر من ملف `.env.example` بنفس الاسم، بهذه القيم:
   - `PORT` = `3000`
   - `JWT_SECRET` = اكتب أي جملة عشوائية طويلة (مثال: `nahj2026SecretKeyXyz987`)
   - `FIREBASE_PROJECT_ID` = افتحه من ملف الـ JSON الذي نزّلته في المرحلة 1 (القيمة أمام `"project_id"`)
   - `FIREBASE_CLIENT_EMAIL` = من نفس الملف (أمام `"client_email"`)
   - `FIREBASE_PRIVATE_KEY` = من نفس الملف (أمام `"private_key"`، انسخها بالكامل بما فيها `-----BEGIN...`)
   - `FIREBASE_DATABASE_URL` = الرابط الذي حفظته في الخطوة 11 من المرحلة 1
   - `WORK_ZONE_LAT` و `WORK_ZONE_LNG` = إحداثيات مدينتك (ابحث بجوجل: **"اسم مدينتك" coordinates**)
   - `WORK_ZONE_RADIUS_METERS` = `15000`
   - `INACTIVITY_ALERT_SECONDS` = `60`
   - `LOW_BATTERY_PERCENT` = `15`
6. اضغط **Create Web Service** بالأسفل
7. انتظر 3-5 دقائق حتى ترى في السجل (Logs) عبارة: `✅ خادم نهج للتوصيل يعمل على المنفذ`
8. بالأعلى، ستجد رابط خادمك جاهزاً، شبيه بـ: `https://nahj-backend.onrender.com`
   افتحه في المتصفح وأضف له `/health` هكذا: `https://nahj-backend.onrender.com/health`
   إن ظهرت لك كلمة `"status":"ok"` فالخادم يعمل بنجاح! 🎉

✅ **انتهت المرحلة 3.** احفظ رابط الخادم، ستحتاجه بالمرحلة القادمة. أخبرني "انتهيت من المرحلة 3".

# المرحلة 4: إنشاء أول حساب مشرف لك

هذه الخطوة تحتاج تشغيل أمر واحد فقط، عبر أداة اسمها **Shell** موجودة داخل Render نفسه (بدون تثبيت أي شيء على جهازك):

1. داخل صفحة خدمتك على Render (`nahj-backend`)، ابحث عن تبويب باسم **Shell** أعلى الصفحة واضغط عليه
2. ستفتح لك نافذة سوداء (Terminal) — اكتب هذا الأمر بالضبط ثم اضغط Enter:
   ```
   node scripts/createFirstAdmin.js
   ```
3. سيظهر لك: اسم المستخدم `admin` وكلمة المرور `ChangeMe123!` — احفظهما، وسنغيّرهما لاحقاً من داخل النظام

✅ **انتهت المرحلة 4.** أخبرني "انتهيت من المرحلة 4".

# المرحلة 5: نشر لوحة التحكم على الإنترنت (Netlify)

1. اذهب إلى: `https://app.netlify.com` → أنشئ حساباً (يفضّل بنفس حساب GitHub، زر **Sign up with GitHub**)
2. **قبل الرفع**: افتح ملف `admin-dashboard/js/config.js` من المشروع (بأي محرر نصوص بسيط، حتى Notepad يكفي)
   وغيّر السطرين إلى رابط خادمك من المرحلة 3:
   ```
   const NAHJ_API_URL = 'https://nahj-backend.onrender.com/api';
   const NAHJ_SOCKET_URL = 'https://nahj-backend.onrender.com';
   ```
   احفظ الملف (Save)
3. في صفحة Netlify الرئيسية بعد الدخول، ستجد مربعاً مكتوباً فيه **"Drag and drop your site output folder here"**
4. اسحب مجلد `admin-dashboard` بالكامل من جهازك وأفلته داخل هذا المربع
5. انتظر ثوانٍ، وستظهر رسالة نجاح مع رابط لموقعك، شبيه بـ: `https://random-name-123.netlify.app`
6. افتح الرابط + `/login.html` في المتصفح، مثال: `https://random-name-123.netlify.app/login.html`
7. سجّل الدخول بالحساب من المرحلة 4 (`admin` / `ChangeMe123!`)

🎉 **الآن لوحة تحكمك تعمل فعلياً على الإنترنت من أي مكان!**

✅ أخبرني "انتهيت من المرحلة 5".

# المرحلة 6: تجهيز تطبيق الأندرويد (APK) للمناديب

هذه أطول مرحلة تقنياً، لكنها تُنفَّذ **مرة واحدة فقط**. بما أنك قلت إنك مستعد ومتحمس، سنجربها معاً؛ إن تعثرت في أي أمر، أرسل لي نص الخطأ بالضبط وسأصلحه.

### أ) تثبيت الأدوات المطلوبة على الكمبيوتر

1. ابحث في جوجل عن: **"Flutter SDK install windows"** (أو mac إن كان جهازك ماك) وادخل للموقع الرسمي `docs.flutter.dev`
2. حمّل ونفّذ التثبيت حسب تعليمات الموقع الرسمي (يشرحها بالصور خطوة بخطوة)
3. ابحث في جوجل عن: **"Android Studio download"** وثبّته أيضاً (يحتاجه Flutter لبناء تطبيقات أندرويد)
4. بعد التثبيت، افتح برنامج **Terminal** (على ماك) أو **Command Prompt / PowerShell** (على ويندوز) واكتب:
   ```
   flutter doctor
   ```
   هذا الأمر يفحص تثبيتك ويخبرك إن كان ناقصاً أي شيء (سيظهر بعلامة ❌ إن كان هناك نقص، مع تعليمات لإصلاحه — نفّذها بالترتيب)

### ب) بناء التطبيق

5. افتح Terminal/Command Prompt وانتقل لمجلد التطبيق، مثال:
   ```
   cd Downloads/nahj-delivery/mobile-app
   ```
6. نفّذ:
   ```
   flutter create . --platforms=android
   ```
   (هذا يولّد ملفات أندرويد الأساسية المفقودة تلقائياً)
7. افتح ملف `android/app/src/main/AndroidManifest.xml` بمحرر نصوص، وأضف الأسطر الموجودة
   في ملف `android/AndroidManifest_permissions_reference.xml` (انسخها قبل وسم `<application>`)
8. عدّل ملف `lib/services/api_service.dart`، غيّر السطر:
   ```
   static const String baseUrl = 'http://YOUR_SERVER_IP:3000/api';
   ```
   إلى رابط خادمك الحقيقي:
   ```
   static const String baseUrl = 'https://nahj-backend.onrender.com/api';
   ```
9. نفّذ:
   ```
   flutter pub get
   ```
   (يحمّل كل المكتبات المطلوبة، قد يأخذ دقيقة أو دقيقتين)
10. أخيراً، نفّذ أمر البناء:
    ```
    flutter build apk --release
    ```
    سيأخذ عدة دقائق أول مرة. عند الانتهاء بنجاح سيظهر لك مسار الملف الناتج:
    ```
    build/app/outputs/flutter-apk/app-release.apk
    ```

### ج) تثبيته على جوال المندوب

11. انسخ ملف `app-release.apk` إلى جوال أندرويد (عبر واتساب لنفسك، أو كيبل USB، أو رابط تحميل من جوجل درايف)
12. على الجوال، افتح الملف — سيظهر تحذير **"لا يمكن التثبيت من مصادر غير معروفة"**، اذهب للإعدادات
    واسمح بـ **"تثبيت من مصادر غير معروفة / Install unknown apps"** لتطبيق الملفات الذي تستخدمه
13. ثبّت التطبيق، افتحه، اختر لغتك، وسجّل دخول بأي مندوب أنشأته من لوحة التحكم

✅ **إن نجحت هذه الخطوة، مبروك — نظامك يعمل بالكامل من الصفر إلى التشغيل الفعلي! 🎉**
